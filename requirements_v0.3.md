# ワイン記録・照会PWA 要件定義 v0.3(実装着手版)

作成日: 2026-07-08 / 対象: 個人専用(単一ユーザー) / 実装: Claude Code

**v0.2からの変更点:**
1. SAT語彙を二層設計に確定(内部=英語canonicalキー、表示=日本語辞書マッピング・編集可)
2. スパークリング拡張項目を追加
3. 劣化(フォールト)チェックリストを新設(自由記述の前段に配置)
4. ドシエ生成プロンプト方針を確定(用語解説なし・技術パラメータ直行)
5. 残オープン事項をすべて確定 → 実装可能状態

---

## 1. 目的

1. 飲んだワインのテイスティング記録(WSET L3 SAT準拠、劣化トラッキング、コンテキスト付き)
2. ワイン名からの照会:生産者・栽培・醸造・熟成スタイル、ヴィンテージ特性、評価(LLM+Tier制限定Web検索)

**ユーザープロファイル前提: WSET Level 3保持・上級の飲み手。アプリ内のあらゆる出力で専門用語の解説・パラフレーズは行わない。**

## 2. アーキテクチャ

| 項目 | 決定 |
|---|---|
| 形態 | PWA(ホーム画面追加、モバイルファースト) |
| ストレージ | IndexedDB(端末ローカル完結、純テキストのみ) |
| 照会エンジン | Anthropic API(claude-sonnet-4-6)+ Web検索(`allowed_domains`強制) |
| Vision | 同API(リスト読み取り、画像非保存) |
| APIキー | 設定画面入力、端末ローカル保存 |

## 3. データモデル

```
Wine(マスター)……… 1銘柄=1レコード。ドシエをキャッシュ
 ├─ TastingRecord …… SAT + 劣化 + コンテキスト
 └─ CellarItem ……… 在庫(本数)
```

### Wine
- id, ワイン名, 生産者, 産地, 品種, タイプ(red/white/rosé/sparkling/fortified), ドシエ(キャッシュ+生成日+出典+信頼度ラベル), 登録日

### TastingRecord
- wine_id, ヴィンテージ, 飲用日, SATフィールド(§4.1), スパークリング拡張(§4.2), 劣化チェック(§4.3), コンテキスト(場所・店名, 同席者), 自由コメント

### CellarItem
- wine_id, ヴィンテージ, 本数

### 語彙アーキテクチャ(二層設計・確定)
- **内部データ: 英語canonicalキー**(例: `blackcurrant`, `brioche`, `wet_stone`)。検索・集計・将来の統計はすべて英語キーで実行
- **表示層: 日本語辞書マッピング**(`blackcurrant → カシス`)。辞書は設定画面で編集可能。訳語変更が過去データに一切影響しない
- 表示は日本語、括弧内に英語キー併記のトグルを設定に用意

## 4. 機能要件

### F1: テイスティング記録

**4.1 SAT構造(WSET L3準拠)**

| 区分 | 項目(内部キーは英語) |
|---|---|
| Appearance | clarity / intensity / colour |
| Nose | condition / intensity / aroma characteristics(1st/2nd/3rd) / development |
| Palate | sweetness / acidity / tannin / alcohol / body / flavour intensity / flavour characteristics / finish |
| Conclusions | quality(BLIC) / readiness for drinking |

- アロマ/フレーバー語彙はL3レキシコンをクラスタ階層(floral / green fruit / citrus / stone fruit / tropical / red fruit / black fruit / herbaceous / oak / yeast(autolysis) / malolactic / oxidative / fruit development 等)で収録
- UI: クラスタ→展開の階層表示 + 使用頻度による並び替え学習

**4.2 スパークリング拡張(タイプ=sparklingの記録で追加表示)**

- mousse: texture(delicate/creamy/aggressive)+ persistence
- autolytic character: 強度(brioche/toast/pastry系はレキシコン側に収録済み)
- dosage perception: brut nature〜doux の体感スケール(ラベル表記とは独立した官能評価)

**4.3 劣化チェックリスト(自由記述の前段に配置)**

| キー | 表示 |
|---|---|
| oxidation | 酸化 |
| reduction | 還元 |
| heat_damage | 熱劣化 |
| premox | PMO(プレマチュア酸化) |
| tca | TCA(ブショネ) |
| lightstrike | 光劣化 |
| brett | ブレタノマイセス |
| va | 揮発酸 |

- 各項目に強度3段階: 軽微 / 明確 / 欠陥レベル
- Nose の condition(clean/unclean)と連動: unclean選択時にチェックリストへ誘導
- **設計意図: 同一銘柄タイムライン(F5)と組み合わせ、生産者×ヴィンテージ単位のフォールト発生率追跡を可能にする(v2統計の主要フック)**

**4.4 2段階入力モデル(確定)**
- **クイック入力: ワイン名のみで保存可**。他フィールドは全てnullable。飲用日は自動、場所・一言メモは任意
- **構造化追記**: 下書きバッジ付き記録にSATフォームを追記
- **同席者: 過去入力からのサジェスト方式**(タップ選択+新規入力)

### F2: ワイン照会(ドシエ生成)

**4.5 出力構成(優先順)**
1. 生産者プロファイル・栽培/醸造/熟成スタイル
2. ヴィンテージ特性(指定時)
3. 評価スコア(媒体名・出典明示)
4. 価格帯・入手性(簡易)

**4.6 生成プロンプト方針(確定)**
- 用語解説・パラフレーズ一切なし。WSET L3+を読者前提とする
- 技術パラメータに直行: 栽培(面積・樹齢・農法認証・収量)、醸造(発酵容器・全房比率・MLF・酵母)、熟成(容器・新樽比率・期間・バトナージュ)、スパークリングは加えてベースヴィンテージ構成・リザーヴワイン比率・ティラージュ期間・ドザージュ量(g/L)
- 情報がソースにない項目は「不明」と明記(推測で埋めない)

**4.7 情報ソースポリシー(Tier制ホワイトリスト)**

| Tier | ソース | 用途 |
|---|---|---|
| 1 | 生産者公式、日本正規輸入元公式(ラック、ファインズ、エノテカ 等) | 栽培・醸造・熟成の一次情報 |
| 2 | Wine Advocate / Vinous / Jancis Robinson / Decanter / James Suckling | 評価・TN |
| 3 | 地域生産者団体公式(CIVC、BIVB 等)、公式ヴィンテージチャート | ヴィンテージ・産地 |
| 4 | Wine-Searcher | 価格・入手性のみ |
| 除外 | コミュニティ評価(Vivino等)、個人ブログ、EC商品説明文 | — |

- 固定ホワイトリストは設定画面で編集可能。具体ドメインは実装時確定

**4.8 検索パイプライン(3ステップ)**
1. 公式ドメイン特定(オープン検索、内容非取得)
2. 本照会: `allowed_domains` = 固定ホワイトリスト + Step 1公式ドメイン
3. フォールバック: 不足要素のみオープン検索、信頼度ラベル「参考(ホワイトリスト外)」+出典必須

**4.9 キャッシュ**
- Wineマスターに保存(出典・信頼度込み)。再照会はキャッシュ+「更新」ボタン。記録作成画面からワンタップ起動

### F3: リスト読み取り(Vision)

- リスト/ラインナップ写真 → 銘柄名・ヴィンテージ・価格を抽出 → 一覧表示。写真は処理後破棄
- 各行アクション: 照会 / 記録作成 / セラー追加

### F4: セラー在庫

- 銘柄・ヴィンテージ・本数のみ。記録作成時「セラーから1本消費」→自動減算。在庫一覧から記録・照会へ直接遷移

### F5: 検索・閲覧

- 全文検索(コメント・自由記述横断)。英語キー・日本語表示語のどちらでもヒット
- フィルタ: 産地 / 品種 / タイプ / 期間 / 場所・店名 / 同席者 / **劣化項目**
- ワイン別タイムライン: 同一銘柄の飲用履歴+フォールト履歴を時系列表示

## 5. 非機能要件

- オフライン: 記録・閲覧・検索はオフライン可。F2/F3のみオンライン必須
- データ保全: JSONエクスポート/インポート(共有シート経由)
- UI: 日本語、モバイルファースト、ダークモード
- PWA: manifest + Service Worker

## 6. スコープ外(v1)

- 写真のDB保存全般 / ラベルからの銘柄自動特定
- 統計・傾向分析(v2候補: フォールト発生率、産地/品種分布。データ構造は対応済み)
- 購入価格・飲み頃ウィンドウ / マルチデバイス同期

## 7. 実装時タスク(要件は確定済み)

1. 固定ホワイトリストの具体ドメインリスト作成
2. L3レキシコン英語キー全量+日本語訳辞書の初期データ作成
3. ドシエ生成・リスト読み取りの各プロンプト実装
